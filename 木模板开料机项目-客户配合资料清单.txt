木模板开料机G-code生成程序开发项目
客户配合资料清单

项目编号：WMB-2025-001
文档版本：V1.0
创建日期：2025年10月16日


一、现有套料系统对接信息

1.1 套料软件基本信息
- 当前使用的套料软件名称及版本号
- 套料结果输出文件的保存路径和命名规则
- 输出文件格式（DXF/JSON/其他）

1.2 样本文件提供（重要）
请提供以下实际套料结果文件：
- 简单矩形件套料文件 2-3个
- 包含开槽的复杂零件套料文件 3-4个  
- 包含圆孔的零件套料文件 2-3个
- 异形件（曲线轮廓）套料文件 2-3个
- 每种文件对应的零件图纸或加工要求说明

1.3 数据格式说明
- 坐标单位定义（毫米/英寸）
- 零件编号、材料厚度等关键字段的具体命名
- 如有数据字段说明文档，请一并提供

二、机床设备详细规格

2.1 设备基本信息
- 四工序开料机品牌和具体型号
- 数控系统类型（如：宝元LNC-M220i、新代21MA等）
- 系统软件版本号
- 机床技术手册或操作说明书（PDF格式）

2.2 G代码规范文档（必需）
- 数控系统支持的G代码指令手册
- M代码、T代码完整列表
- 特殊指令或非标准代码说明
- 如无完整手册，请提供常用指令示例

2.3 刀具配置信息
请按以下格式提供四把刀具的详细参数：

刀具位置 | 刀具类型 | 刀具直径 | 主轴转速范围 | 进给速度范围 | 最大切削深度 | T代码
T01      | 开料刀   |         |            |            |           |
T02      | 开槽刀   |         |            |            |           |
T03      | 钻孔刀   |         |            |            |           |
T04      | 倒角刀   |         |            |            |           |

2.4 加工工艺标准
- 不同厚度板材的标准切削参数
- 刀具切换的标准指令序列
- 安全高度、快进高度的设置值
- 切入切出方式的企业标准

三、网络通信环境

3.1 网络配置
- 机床控制器的IP地址
- 网络连接方式（有线/无线）
- 是否使用固定IP还是DHCP自动分配
- 网段和子网掩码信息

3.2 文件传输方式
- 支持的文件传输协议（FTP/SFTP/共享文件夹/USB）
- 如果是网络传输，请提供：
  * 服务器地址和端口号
  * 用户名和密码（如需要）
  * 文件存储的目标文件夹路径

3.3 通信测试准备
- 请安排网络管理员确认防火墙设置
- 提供一个简单的NC程序文件，我们先测试传输功能
- 确认机床能否正常读取外部NC文件

四、软件运行环境

4.1 客户端电脑配置
- 操作系统版本（Windows 10/11 具体版本号）
- CPU型号和内存大小
- 显卡信息（影响3D仿真效果）
- 硬盘可用空间

4.2 软件环境
- 是否安装了杀毒软件（可能影响程序运行）
- 用户权限等级（建议使用管理员权限）
- 是否有其他工业软件需要兼容运行

4.3 现有软件版本
- 套料软件的安装路径
- 是否需要我们的程序与套料软件直接集成
- 其他相关CAD/CAM软件信息

五、验收测试准备

5.1 测试用例
请准备5-10个标准测试案例，包含：
- 测试用的套料结果文件
- 对应的零件图纸
- 预期的加工结果和质量要求
- 加工时间和精度标准

5.2 验收标准明确
- "单板NC生成时间<3秒"是基于多少个零件的复杂度？
- 仿真预览的精度要求（是否需要显示刀具轨迹细节？）
- 程序下发的成功率要求（99%以上？）

5.3 测试人员安排
- 指定懂机床操作的技术人员作为项目对接人
- 安排测试时间窗口（建议连续3天，每天2-4小时）
- 提供技术人员的联系方式（手机、微信、QQ）

六、技术支持配合

6.1 远程协助准备
- 在测试电脑上安装TeamViewer或向日葵远程控制软件
- 确保网络环境支持远程连接
- 安排测试期间的技术人员在线时间

6.2 应急联系方式
- 项目技术负责人姓名和联系电话
- 设备厂家技术支持联系方式（如遇设备问题）
- 紧急情况下的备用沟通渠道

6.3 问题反馈机制
- 测试过程中如出现问题，请详细记录：
  * 操作步骤
  * 错误提示信息
  * 系统环境状态
  * 可能的截图或日志文件

七、项目配合时间节点

7.1 资料提供阶段（开发启动前）
- 上述技术资料请在项目启动后3天内提供完整
- 样本文件和设备手册是开发的基础，请优先提供

7.2 中期验证阶段（开发中期）
- 提供中间版本供客户测试数据解析功能
- 验证G代码格式和设备兼容性
- 确认界面设计和操作流程符合使用习惯

7.3 最终测试阶段（交付前）
- 安排连续3天的集中测试时间
- 提供实时技术支持和问题解决
- 完成验收标准确认和使用培训

八、特别说明

由于我们无法直接接触您的机床设备进行现场调试，以上信息的准确性和完整性直接关系到项目的成功交付。如果在资料收集过程中遇到困难，请及时沟通，我们可以协助制定替代方案。

为确保项目顺利进行，建议：
1. 安排专门的技术对接人员负责资料整理
2. 如有疑问及时沟通，避免理解偏差  
3. 保持测试环境的稳定性，避免测试期间变更配置
